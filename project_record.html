雙軌高防護爬蟲管線設計

本專案針對動態與靜態兩種不同架構的網頁，設計了專屬的數據採集路徑。我們秉持「以最少伺服器開銷換取最高完整度」的原則進行逆向工程。

步驟 1：逆向分析 JS 封包 (來源一：工業館)
跳過網頁渲染層，透過 Chrome DevTools 網路封包解碼，挖掘出底層的公開 Google Sheet 密鑰（ID：18Hq6sUrweHmm_08AcFPBnQ8XypXJ7CM6）。
步驟 2：直連 API 匯出無損 CSV (來源一)
利用 Google Visualization 接口一鍵下載無損 CSV 數據，利用 Python 內建的 CSV 引擎，精準處理單個儲存格內包含換行符號的特殊格式，成功取得 79 筆數據。
步驟 3：分頁遍歷與第一層卡片爬取 (來源二：智慧轉型)
設定行業別參數 mindusty=-1 以獲取全量工具，在 9 個分頁中解析 HTML 節點，搜集基本屬性與 145 個方案細節頁的動態連結參數。
步驟 4：深度解析內頁以抓取敏感欄位 (來源二)
遍歷請求 145 個內頁，爬取「方案規格說明」、「成功案例效益」等非結構化大段文字，並利用 Regex 正則分析，抓取廠商統一編號（統編）、聯絡郵件等關鍵資訊。

資料清洗與 AI-Ready 欄位設計

為了讓大語言模型 (LLM) 與向量數據庫 (Vector DB) 能夠執行精確硬性過濾（如價格、期程）與高準度語意檢索，我們將凌亂的原始網頁字串清洗為標準化欄位：

欄位名稱 資料類型 清洗與 RAG 應用邏輯
id String 以 moeai_ 與 smebiz_ 為前綴的全球唯一辨識碼，防止資料庫衝突。
provider_tax_id String 台灣企業 8 碼統一編號。便於後續與企業 ERP/CRM 進行廠商關聯。
category_standardized String 分類標準化：將兩邊網站不一致的標籤，統一映射至「行銷與商務推廣」、「人資與知識管理」等 8 大 RAG 優化類別。
price_amount Integer 數值化價格：以正則過濾掉千分位逗號與文字，保存純整數（如 $24,000 -> 24000），供 Vector DB 進行硬性區間過濾。
duration_months Integer 數值化授權月數：從混亂字串（如「使用月數 : 6」、「期程12個月」）中提取純整數，供預算期程篩選。
success_cases String 提取出的成功案例與效益分析。在 RAG 檢索中，這是評估 AI 方案是否適合特定情境的關鍵文字區塊。
source_url String 原始網頁引用來源。提供給 AI 模型，讓生成回覆時能夠附帶「真實連結引用」，消滅幻覺。
